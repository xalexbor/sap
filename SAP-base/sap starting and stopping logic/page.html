Каждая SAP система имеет как минимум одну базу данных.
В Java стеке может быть узел Central Service
В ABAP кофигурации может быть узел ABAP Central Service

<sid>adm - пользователь управления службами SAP. Все процессы и службы запускаются из под него

Ниже отражена последовательность запуска служб на системе SAP:



1. Инстанции без ABAP диспетчера, так называемая Central Service (java message server; abap censtal service)
2. База данных
3. Инстанции с ABAP диспетчером и центральная инстанция
4. Все другие инстанции: Диалоговые инстанции


Остановки происходит в абсолютно обратном порядке. Сначала другие инстанции, центральная инстанция, база данных и central service.

Для управление запусками/остановами служб САП была придумана такая штука, как SAP MC

ЛОГИ: находятся по адресу: /usr/sap/<SID>/<instance><No>/work
  • stderr1..n - логи sapstartsrv, ошибки выполнения
  • sapstartsvr.log - Собственно сами логи
  • sapstart.log - Стартовые логи базы данных,сервера сообщений,диспетчера
  • dev_ms - рабочие логи сервера сообщений
  • dev_disp - рабочие логи диспетчера
  • dev_w0..n - рабочие логи рабочих процессов
