Само внедрение в организацию.

В SAP есть набор основных компонентов. Стандартные возможности ERP системы.
1. Производится адаптация системы под организацию. Т.н. пользовательская настройка. Сюда входят только те данные которые присущи конкретной организации.
2. Работа с репозитарием. Собственно, сама разработка. Различают несколько видов разработок:
  • Внутрифирменная разработка. Собственные объекты, таблицы, транзакции, отчеты и т.д.
  • Расширение клиента. Доработки специфичные для клиента.
  • Модификация. Изменение стандартного кода SAP. Для этого есть специальный ассистент, который отслеживает внесенные изменения в стандартный код SAP.



При развертывании всей системы устанавливаются сразу 3 базы. Образуя т.н. трехсистемный ландшафт:
1. Разработка (DEV)
2. Обеспечение качества (QAS)
3. Продуктивная эксплуатация (PRD)

это будет трехуровневая система ОДНОГО манданта, например, 100.
Для того, чтобы изменения переходили из одной системы в другую существуют ТРАНСПОРТЫ.

Чтобы переносить данные/наработки из одной инстанции в другую руководитель разработок проектов создает процесс запроса на изменение.  В котором оператор раскидывает позадачно каждому сотруднику необходимые для конкретной разработки задачи. Запрос закрывается, когда все задачи закрыты.

Чтобы внедрить SAP в организацию, либо уже внедрить необходимый функционал в существующую систему необходимо пройти след. этапа:
1. Анализ и проектирование
2. Внедрение
3. Тестироване
4. Администрирование


ALE - Application Link Enabling - средство для создания распределенной приложения. Не системы SAP, а логически сформированную на отделах организации. Позволяет обмениваться данными между узлами в такой системе.

BAPI - интерфейс бизнес-приложения.
RFC - удаленный вызов функций. Интерфейсный протокол SAP. Основанный на протоколах CPI-C и TCP/IP. Применяется для вызова уже определенных функций в удаленной или в той же системе.
